# enterprise-ayx-lab

本仓库用于归档和发布多个独立的 HTML 页面，作为企业内部实验与资源展示的集中存放地。所有页面均为静态 HTML 文档，不依赖特定域名或后端服务，适合直接通过 GitHub Pages 或其他静态托管服务访问。

## 项目简介

enterprise-ayx-lab 是一个轻量级的页面归档仓库，旨在收集、整理并持续发布各类独立 HTML 页面。这些页面可涵盖内部工具演示、技术原型、文档展示、实验性交互界面等场景。仓库不绑定任何具体业务或品牌，保持通用性与可扩展性。

## 目录说明

仓库根目录结构如下：

```
enterprise-ayx-lab/
├── index.html          # 仓库首页（可选）
├── pages/              # 存放所有独立 HTML 页面
│   ├── page-1.html
│   ├── page-2.html
│   └── ...
├── assets/             # 公共资源（CSS、JS、图片等）
│   ├── css/
│   ├── js/
│   └── images/
├── docs/               # 文档或说明文件（可选）
└── README.md           # 本文件
```

- `pages/`：主要页面目录，每个 HTML 文件对应一个独立页面。
- `assets/`：存放页面共用的样式、脚本、图片等资源文件。
- `docs/`：可选，存放与项目相关的文档或使用说明。

## 页面归档说明

- 所有页面均为纯前端实现，无需后端支持。
- 页面之间相互独立，无强依赖关系，可单独访问。
- 页面内容应保持简洁、清晰，避免包含过多外部依赖或追踪代码。
- 如有新增页面，请放入 `pages/` 目录，并确保文件名唯一。

## 维护说明

- 本仓库由维护者不定期更新，新增或修改页面时请保持目录结构一致。
- 建议在提交新页面时附上简要说明（可写在 commit message 中）。
- 若页面需要引用外部资源，请优先使用本地 `assets/` 目录下的文件，减少对外部链接的依赖。
- 仓库默认启用 GitHub Pages 功能，可通过设置指定发布目录（如 `/` 或 `/docs`）进行访问。

## 贡献

欢迎通过 Issue 或 Pull Request 提出建议或提交新页面。提交前请确保页面内容符合仓库定位，且不包含敏感或违规信息。
